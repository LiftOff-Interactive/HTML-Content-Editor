Overview

What is Claude Code?

Most people know Claude as a chatbot in a browser. Claude Code is something different — it works inside your computer, inside your project folders, building real files that you can open, test, and deploy.

Claude.ai (browser)
Great for thinking and drafting
You chat, it responds. Outputs stay in the browser window. You have to manually copy anything useful. Good for ideation and writing — not for building complete files.
Claude Code (terminal)
Builds real files in your folder
You give it a brief. It writes HTML, CSS, JavaScript, SCORM packages — whatever you need — directly into your project folder. Open the files in a browser and test them immediately.

What can it build for eLearning?

  • Complete HTML/JavaScript eLearning modules — interactive, multi-screen, with quizzes
  • SCORM 1.2 and 2004 packages ready to upload to any LMS
  • Brand-consistent templates driven by your own spec file
  • Accessibility-audited content meeting WCAG 2.2 AA standards
  • PDF job aids, study guides, and reference materials
~$1 Typical API cost per module build on pay-as-you-go credits
60s Average time to build a full 10-screen HTML module from scratch
100% Offline-capable — modules work without internet once built
What subscription do I need?

Claude Pro ($20/month) includes enough usage for regular module builds and is the best starting point. Claude Max ($100/month) suits L&D professionals building daily at volume. Alternatively, API pay-as-you-go charges per token with no subscription — typically under $2 per module depending on length. All three work with Claude Code.

How this stacks up against traditional authoring tools

Articulate 360 costs ~$1,299/year. Adobe Captivate runs ~$408/year. Both require weeks of training before you can build anything credible. With Claude Code on a Claude Pro subscription ($20/month), you can build production-ready modules from week one — and on pay-as-you-go API pricing, the typical cost per module is under $2 regardless of length or complexity. For an L&D professional or solo consultant, that’s a difficult argument to make against.

Claude Code vs the Claude Desktop App

Claude Desktop App
Conversations, not files
The desktop app is Claude in a window. You can chat, get answers, and connect MCP servers — but it doesn’t write files directly into your project folders or run build commands on your behalf. Output stays in the conversation window. You still have to copy and paste anything useful.
Claude Code (CLI)
Builds inside your project
Claude Code runs in your terminal, inside your actual project directory. It creates, reads, edits, and deletes real files. It runs scripts. It sees your full codebase context. Every build goes directly into the folder you’re working in — no copy-paste step.
The practical difference for eLearning professionals

With the desktop app you’re still doing the assembly yourself: copy Claude’s output, paste into a file, save, test, repeat. With Claude Code, there’s no handoff — it writes the HTML module directly, runs the audit script, and packages the SCORM ZIP. You go from brief to deployable file without touching a text editor. That’s not a chatbot helping you build. That’s a co-developer building alongside you.

Installation

Getting Set Up

Setup takes about 10 minutes and only needs to be done once. You need two things: Node.js (a free engine) and a Claude account. Everything else is a few commands in your terminal.

Five steps to your first build

1
Install Node.js
Go to nodejs.org, download the LTS version, run the installer. Click through defaults. No admin access? Use nvm (Node Version Manager) — it installs to your user profile with no IT password needed.
2
Open your terminal
Mac: Cmd+Space → type “Terminal” → Enter.   Windows: Windows key → type “PowerShell” → Enter. The terminal is just a text window — you only need it for setup.
3
Install Claude Code
Paste this command and press Enter:
npm install -g @anthropic-ai/claude-code
4
Log in
Type claude and press Enter. The first time, a browser window opens — sign in with your Claude.ai Pro or Max account. One-time step.
5
Navigate to your project folder and start
Mac: type cd  then drag your folder into Terminal.   Windows: Shift+right-click your project folder → Open in Terminal. Then type claude. You’re ready to build.
No admin access at work?

Use nvm on Mac or nvm-windows on Windows. Both install Node.js entirely inside your user profile — no IT password needed. Once Node is installed this way, the Claude Code install also requires no admin rights. See the full guide at e-fuse.com for step-by-step instructions.

The Art of the Brief

Prompt Smarter,
Build Better

Claude Code’s output is only as good as your brief. A vague prompt produces a generic result. A specific, structured prompt produces production-ready output. The difference is remarkable — and learnable.

The single best investment you can make in Claude Code is learning to write a great brief. It takes 2 extra minutes and saves hours of revision.
e-Fuse Practice Note

Six rules for better prompts

Tap each card to reveal the detail

Rule 1
Lead with a reference file
Tap to reveal ›
Always start with “Read CLAUDE.md” or “Read template.html”. This grounds every build in your standards without re-typing them. Your spec file does the work.
Rule 2
Be specific about numbers
Tap to reveal ›
Don’t say “a few screens” — say “10 screens”. Don’t say “a quiz” — say “5 multiple-choice questions, 4 options each, with per-answer feedback and a 4/5 pass threshold”.
Rule 3
Use hex codes, not colour names
Tap to reveal ›
“Navy blue” could be anything. “#0D224F” is unambiguous. Same for fonts — say “Overpass, fallback Arial” not just “a clean sans-serif”.
Rule 4
Iterate, don’t over-specify
Tap to reveal ›
Build a rough first draft, review it in the browser, then refine with follow-ups. Trying to specify everything in one prompt often backfires — Claude performs better with focused, incremental instructions.
Rule 5
Ask Claude to review its own work
Tap to reveal ›
After a build: “Review what you just built. Check for broken navigation, missing quiz feedback, and any errors. Fix anything you find.” This catches most bugs before you even open the browser.
Rule 6
Anchor edits to specific locations
Tap to reveal ›
Instead of “make the text better”, say “Rewrite screen 4 to be shorter — max 60 words, same key messages.” Specificity prevents Claude from changing things you didn’t want changed.

Design Workflow

Design First,
Build Fast

Claude.ai (the browser version) and Claude Code (the terminal) are a powerful pair. Design visually in the browser, build functionally in the terminal. Each tool does what it does best.

The design-to-production pipeline

1
Design in Claude.ai
Open claude.ai in your browser. Describe the visual design for your module — brand colours, layout, components. Ask Claude to render it as HTML so you can preview it instantly.
“Design a professional eLearning template. Navy #0D224F, accent #00AEEF. Fixed header, progress bar, card content area, clean nav buttons. Show as HTML.”
2
Refine visually in the browser
Claude renders your design as an interactive artifact. Chat to refine: “Make the cards wider”, “Change the font”, “Add a sidebar”. Keep iterating until it looks right.
3
Copy the code and save it
Click “Copy code” on the artifact. Save it as template.html in your project folder. This is now your design reference file.
4
Build on it in Claude Code
Open Claude Code in your project folder and say:
“Read CLAUDE.md and template.html. Build a full eLearning module on [topic] using template.html as the visual design. Keep layout and branding exactly as-is.”
Why branding matters in eLearning

Branded eLearning signals quality and builds trust. Learners notice when content feels deliberate and polished — and they notice when it doesn’t. Encode your brand once in CLAUDE.md and every module automatically matches. That’s the difference between generic AI output and a professional product.

Got legacy SCORM content?

Convert old modules into fresh, redesigned ones

Drop a SCORM ZIP file directly into your project directory, then tell Claude Code to extract it and rebuild it to your current standards. Claude reads the existing content structure, applies your CLAUDE.md brand rules, and outputs a modern branded module. A decade of visually outdated eLearning can be relaunched in under an hour.

“There’s a SCORM file called legacy-module.zip in this folder. Extract its content, then rebuild it as a new module following CLAUDE.md standards.”

The design.md approach: a portable spec file

Create a design spec in Claude.ai — drop it straight into Claude Code

1
Design in Claude.ai, then export as Markdown
After designing and refining your template in Claude.ai, ask it to export all design decisions as a Markdown spec file. Not code — a plain-language document describing every rule.
“Summarise every design decision we’ve made in this conversation — colours, fonts, spacing, component styles, layout rules — as a design.md file I can drop into a project folder.”
2
Save design.md in your project folder
Drop the exported design.md file into your Claude Code project directory alongside CLAUDE.md. It lives next to your source files — version-controllable, human-readable, and ready for Claude Code to use on every session.
3
Reference it in every build prompt
Open Claude Code in your project folder and instruct it to read both files before building. The spec file acts as a persistent visual brief across every session, even after weeks away from the project.
“Read CLAUDE.md and design.md. Build a full eLearning module on [topic]. Apply every design rule in design.md exactly.”
Why Markdown beats template.html for design specs

A template.html file is code — useful as a starting point, but opaque and hard to update selectively. A design.md file is plain language: open it, read it, hand it to a client, update a single rule without touching HTML. Claude Code reads both equally well. The spec-file approach also scales across projects and teams in a way raw HTML templates never could. Your design decisions become an asset rather than buried code.

Your Most Powerful Tool

The CLAUDE.md
Spec File

A CLAUDE.md file is a plain text file you create in your project folder. Claude Code reads it automatically every session in that folder. It encodes your standards once so you never have to repeat them.

Without CLAUDE.md
“Build a 10-screen module on workplace communication. Use navy #0D224F and blue #00AEEF. Font is Overpass. Include a progress bar. 80 words per screen max. Australian English. SCORM 1.2 compatible. Quiz with 5 questions, 4/5 pass. Completion screen at the end...”
With CLAUDE.md
“Read CLAUDE.md, then build a module on workplace communication.”

The CLAUDE.md file handles everything else. Here’s what one looks like:

# My eLearning Module Standards ## Brand - Primary: #0D224F (navy), #00AEEF (accent blue) - Font: Overpass, fallback Arial - Logo: embed from logo.png as base64 - Always include a progress bar ## Structure - 10 screens: 1 title + 8 content + 1 quiz - Quiz: 5 questions, 4/5 pass, per-answer feedback - End with a completion screen ## Technical - Single HTML file, no external dependencies, offline-capable - SCORM 1.2 compatible ## Content - Australian English spelling - Conversational but professional tone - Max 80 words per content screen
Build a library of spec files

Once you have a CLAUDE.md that works well, save copies for different clients, brands, or module types. You’re building a reusable eLearning production system — not just a one-off file. Each new project inherits all your best practices instantly.

Advanced Tools & Workflows

Once you’re comfortable with the basics, these tools turn Claude Code from a great build assistant into a full eLearning production system.

Version Control
GitHub
Stores every version of your project files. Roll back to any previous build in one click. Your whole team can access and collaborate on the same files. Use GitHub Desktop (desktop.github.com) for a visual interface — no command line needed.
IDE Integration
VS Code
Free code editor that pairs perfectly with Claude Code. Open your project, use the built-in terminal (Ctrl+`) to run Claude Code, and watch files update in real time. Install the Claude Code extension from the VS Code marketplace for a full sidebar experience.
Extend Claude Code
Skills & Plugins
Claude Code supports skills — specialist extensions that add slash commands for specific domains. Browse available skills in the Claude Code marketplace (claude.ai/code) and install from the desktop app. Type /skills inside Claude Code to see what’s already installed.
Knowledge Management
Obsidian
Free note-taking app that stores everything as plain files. Paired with the wiki-brain skill, Claude Code builds and maintains a linked knowledge base of everything you learn and build. Use /wiki-brain ingest [file] to turn SME documents into searchable knowledge.

Going further: MCP servers

Connect Claude Code directly to your tools via MCP

Model Context Protocol (MCP) servers let Claude Code connect to external systems — your Google Drive, Notion workspace, SharePoint library, GitHub repos, or any REST API. Instead of copy-pasting source content into your project, Claude reads it live from the original location and uses it in the build.

Set up an MCP server in Claude Code settings (/mcp), or ask Claude to set one up for you: “Add the Google Drive MCP server so you can read files from my Drive folder.” For eLearning, this means pulling SME content, policy documents, or brand guidelines directly from wherever they already live.

What Claude Code can’t do (yet) — and what to use instead

Limitation
Video Production
Claude Code writes HTML, CSS, and scripts — it can’t produce video files, screen recordings, or animated exports.
Workaround
Canva Video / Loom / Adobe Express
Canva Video and Adobe Express handle motion graphics and branded video. Loom is ideal for screencasts and walkthroughs. Claude Code writes the script; these tools produce the video. Embed the result as an iFrame or linked asset in your module.
Limitation
AI Narration & Audio
Claude Code cannot generate audio files, voiceovers, or text-to-speech narration tracks for your modules.
Workaround
ElevenLabs / Murf
ElevenLabs produces broadcast-quality AI voices from a script — clone your own voice or choose from a library. Murf is designed specifically for eLearning narration with slide-level sync. Claude Code writes the narration script; these tools give it a voice.
Limitation
Live Surveys & Data Collection
Claude Code builds static HTML. It can’t natively submit responses to survey platforms, connect to live LMS databases, or pull real-time data from external APIs.
Workaround
Qualtrics / Google Forms / iFrame Embed
Ask Claude Code to embed a Qualtrics or Google Forms survey as an iFrame inside a dedicated module screen. The survey handles all data collection and reporting; your HTML wraps the learning journey around it. For LMS tracking, pair with a standard SCORM completion trigger.
Limitation
Visual Design Mockups
Claude Code builds what it’s told to build. Without a clear visual brief, output defaults to generic layouts. It can’t show you a preview before it writes the file.
Workaround
Claude.ai (browser) for Design First
Design in Claude.ai first — iterate visually until the layout looks right, then export the design decisions as design.md or template.html and hand it to Claude Code to build on. The browser is your design canvas; the terminal is your build engine. See Screen 5 for the full workflow.

Quality Control

Build Your Own
Quality Audit

AI builds fast — but it makes the same mistakes repeatedly. A quality audit script runs in 2 seconds and catches every common failure before a learner ever sees it. Here’s how e-Fuse does it, and how you can build your own.

What a good audit script checks

1
Screen count consistency
The TOTAL_SCREENS variable declared in the JS matches the actual number of id="screen-N" divs in the HTML. Mismatch = broken progress bar and navigation.
2
Navigation completeness
Every content screen has at least one goTo() call. Screens without navigation leave learners stranded — a common AI oversight on complex modules.
3
Quiz pattern compliance
All inline quiz buttons use onclick="iqAnswer(this)" with data-result and data-feedback attributes. Every data-id has a matching feedback div. No raw apostrophes inside feedback text (they break the onclick handler).
4
Scroll behaviour
window.scrollTo({top:0,behavior:"instant"}) — not smooth, not scrollTo(0,0). The wrong version causes a visible header flash on every screen transition.
5
Asset embedding and branding
Logo is embedded as a base64 data URI (not linked externally — linked logos break when the file is moved or uploaded to an LMS). Tagline and CTA links are present.
Running an audit script takes 2 seconds and catches the bugs that 10 minutes of manual clicking would miss. Make it part of your build workflow — not an afterthought.

Troubleshooting

When Things
Go Wrong

Claude Code is remarkably reliable, but every practitioner hits the same handful of problems. Here’s how to resolve the most common ones without starting over.

Five problems and their fixes

1
Claude stops mid-build
Claude has a context window — on long or complex builds it stops before finishing the file. This is normal. Type “continue” or “keep going from where you left off” and it resumes exactly where it stopped. No context is lost.
2
The module looks broken in the browser
Open DevTools (F12 on Windows, Cmd+Option+I on Mac) and click the Console tab. Read the error message. Most common cause: a raw apostrophe inside a data-feedback attribute. Run python audit_module.py — it catches this in 2 seconds.
3
A permission prompt appears
Claude Code asks before running scripts or accessing files outside your project folder. Type y to approve the action, or press Enter. If you’re working in a trusted local project and want to skip prompts for the session, start Claude Code with the --dangerously-skip-permissions flag — only use this in environments you control.
4
Claude edits something you didn’t want changed
Be specific in your next prompt: “Only change screen 4. Do not modify any other screens or functions.” Specificity prevents Claude from interpreting your instruction broadly. If you use GitHub, roll back with git revert.
5
Node.js or npm not found after installation
Close and reopen your terminal. PATH updates only take effect in new terminal windows. If it still doesn’t work on Windows, search “Edit environment variables” in the Start menu and confirm the Node.js folder is listed in your PATH variable.
The fastest debugging loop

Build → open in browser → spot the issue → describe it to Claude precisely (“Screen 4 quiz feedback is not appearing after selecting an answer”) → Claude fixes it. One or two rounds of this is enough for almost any problem. Resist the urge to start over — iteration is faster.

Still stuck? Just ask Claude

If you get stuck or think you’ve made a mistake, ask Claude directly inside Claude Code. Claude has full knowledge of its own setup process, error patterns, and configuration quirks. You’ll get faster and more accurate help than searching online or posting in a forum.
