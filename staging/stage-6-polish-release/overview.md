# Stage 6 — Polish + Release

## Goal
Get the project ready for public release on GitHub. This is about fit and finish, documentation, and making the tool discoverable and usable by a stranger who finds it on GitHub. No new widgets in this stage — everything new goes post-v1.

## Features in This Stage
- `feature-release.md` — README, GitHub Pages setup, UX polish pass, v1 release checklist

## Definition of Done
- [ ] GitHub repository is public
- [ ] GitHub Pages is enabled and the editor loads at the Pages URL
- [ ] `README.md` is complete with screenshots, feature list, usage instructions, and "how to contribute" section
- [ ] The editor opens without errors in Chrome, Firefox, Safari, and Edge
- [ ] All Tier 1 and Tier 2 widgets are functional
- [ ] Export produces a valid, self-contained HTML file
- [ ] JSON save/load round-trip works correctly
- [ ] No console errors in a clean browser session
- [ ] UX polish: loading states, empty state (new document prompt), tooltips on toolbar buttons
- [ ] A sample exported HTML file is included in the repo as a demo
- [ ] The repo has a clear open-source license (MIT recommended)

## Notes
- Take screenshots of the editor with real content for the README — they matter more than words
- The "demo" exported HTML file should show all 10 widgets so visitors can see what the tool produces
- Post-release: announce on relevant communities (Reddit r/instructionaldesign, r/elearning, LinkedIn) — but that's outside the scope of this project's docs
